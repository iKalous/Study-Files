路径和cd

如PS C:\Users\86139>   那PS 指的是工作环境 后面的是工作路径

cd

cd .\. + 文件名
修改当前工作路径

注意 .. 指的是该工作路径的上级工作路径
cd ..可以修改当前路径为上级工作路径

当然cd指令 使用绝对路径的情况更加普遍

对于两个工作路径

pushd指令可以将当前路径放入堆中

popd则是取堆顶路径，这样就可以避免输入太长的地址出错


约定路径 ~
cd ~ 指的是用户根目录

about 路径

相对路径

输入命令 .\. + 地址名

增删改查

增（make direction）  mkdir + 新建文件名称  （代码d）
（new item） ni + 新文件名 （代码-a）

删（remove）rm + 要删的文件 （注意！！！rm删除文件不可恢复！！！）

查
（list） ls
只输入ls指令会显示当前工作路径下的子文件

显示文件内容
（concatenate） cat + 文件名


改

（move）mv + 待修改文件名 + 修改为文件名（相当于重命名吧）
mv + 文件名 + 新地址（相当于移动）
二者本质上是一样的，但是根据使用的方法不同出现了两种用法



获取帮助
（manu） man + 命令名称
	将会显示该命令的功能

可有用ctrl + ‘r'

ctrl + 'l' 可以把命令行置顶（相当于写文本的时候打一堆回车）
